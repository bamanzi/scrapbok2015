                Matt Briggs fistofsenn • a year ago

                Ok, so things are still in a fairly raw form (no autoloads, some strange defaults that are there for my use and should be in my own .emacs.d, dependancies that are hard to find and not packaged up with it, etc) I also am not really an emacs-the-editor user, cause I use evil-mode exclusively and manually rebind everything I use, so I don't have any default bindings for anything. Finally, I'm a fan of smaller focused libs, so this is spread out over several projects

                railway.el - https://github.com/mbriggs/rai...

                This is pretty much just an extraction of the rinari proj root finder, minor-mode stuff, keymap, and the "enable/disable mode as you navigate to rails projects" thing.

                railgun.el - https://github.com/mbriggs/rai...

                I work on a 200 kloc enterprise rails app for my day job that has been around since rails 1, so a big part of this is being able to gracefully handle broken conventions.

                this depends on
                joseph-file-utils http://www.emacswiki.org/emacs...
                inflections - https://github.com/remvee/emac...
                railway

                it also uses eieio (CLOS implementation from cedet) and ido, so it needs a recent-ish emacs this is the core proj.

                it does
                - find by type (model/controller/etc)
                - find by class (searches everything)
                - create a file based on type (so like rg-create-model will prompt you for a name, then create the file in app/models)
                - toggle test and implementation
                - it will first look for a spec or test where it should be by convention, if that cant be found it searches for all files in the proj that match the current file name with _spec or _test, and asks the user to choose.
                - find blueprint/factory
                - prompt based on model, then open the blueprints or factories file (if you use machinist or factory_girl) and jump to the line that defines the factory for the specified class
                - find schema
                - searches schema.rb or structure.sql for the definition of the specified class
                - everything is cached, so the first time you invoke any finders, it will scan all files, after that it should be instantanious until you clear caches

                All of the searching will be in ruby class notation, so if you are looking for app/models/foo/bar.rb, it will show up as Foo::Bar in the ido completing search

                It also has a configuration data structure that defines file types (like model, controller, etc) and where to find them. This lets you easily add non standard locations, or change where things are searched for. If you add a type, there are macros for creating "find" functions, as well as "create" functions very easily

                rails-assets.el - https://github.com/mbriggs/rai...

                this just gives finders for javascripts and stylesheets folders

                test-js - https://github.com/mbriggs/.em...

                This project isn't rails specific (i use it in node projects too), but provides toggle test and implementation behavior for mocha and jasmine tests, does the same searching behavior as railgun (first look where the test should be, if you cant find it then search for files who are good candidates by name), and has a compilation mode for grunt test tasks

                -----

                All this stuff together works great for me, and I like it a lot more then anything else I have tried in the emacs world so far. Unfortunately, it requires a lot of wiring stuff up, and documentation is very sparse. I've been developing all of this over the last 8 months or so, and for me stuff is pretty stable, so the next step is actually making this stuff ready for public consumption -- documentation, make some better defaults, autoloads, package for elpa, etc

                                    I use devel-which gem ( https://rubygems.org/gems/deve... ) in emacs.

                                    (require 'ffap)
                                    (add-to-list 'ffap-alist '(ruby-mode . ffap-ruby-mode))

                                    ;; C-x C-f at "require 'xxx'" sentence.
                                    (defun ffap-ruby-mode (name)
                                    (shell-command-to-string
                                    (format "ruby -e 'require %%[rubygems];require %%[devel/which];require %%[%s];
                                    print(which_library(%%[%s]))'" name name)))
